ORCINY FRITZING MAPPING GUIDE
=============================

Starter file:
  orciny_wiring_starter.fzz

Why starter and not final exact layout:
  Stock Fritzing installs often do not include exact parts for:
  - Adafruit Feather RP2040
  - Adafruit Feather M0 + NeoPXL8 FeatherWing
  - Adafruit Motor FeatherWing
  So this starter is a valid openable project base. Replace generic blocks with exact parts if available.

Target wiring to implement in the Fritzing project:

1) Controller link
  RP2040 Serial1 TX  -> M0 Serial1 RX
  RP2040 GND         -> M0 GND

2) Switches (momentary to GND, INPUT_PULLUP in firmware)
  SW1 -> RP2040 pin 2 and GND
  SW2 -> RP2040 pin 3 and GND
  SW3 -> RP2040 pin 4 and GND

3) Spark / pulse filaments (via MOSFET drivers)
  RP pin 5  -> Spark 1 driver gate
  RP pin 6  -> Spark 2 driver gate
  RP pin 9  -> Spark 3 driver gate
  RP pin 10 -> Spark 4 driver gate
  RP pin 11 -> Pulse filament driver gate

4) Beam and peltier control (via MOSFET/relay)
  RP pin 12 -> Beam R gate
  RP pin 13 -> Beam G gate
  RP pin 14 -> Beam B gate
  RP pin 15 -> Peltier enable gate/relay input

5) Motor FeatherWing outputs
  Motor port M1      -> DC pump
  Stepper port 2     -> Claw stepper (M3/M4)

6) NeoPXL8 core outputs from M0
  D0 -> Core strip set 1
  D1 -> Core strip set 2
  D2 -> Core strip set 3
  D3 -> Core strip set 4

7) Power rails
  +12V -> motor wing VM, pump, stepper, peltier power stage
  +5V  -> MCU logic and NeoPixel power (if strips are 5V type)
  GND  -> common ground for all supplies, controllers, loads, and switch returns

Safety checks
  - Use flyback diodes where required by motor/pump driver design.
  - Do not drive filaments, peltier, or high-power LED channels directly from MCU pins.
  - Confirm current capacity and thermal design before powering full load.
