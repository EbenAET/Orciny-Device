ORCINY KICAD NET MAP - CURRENT REVISION
=======================================
Updated for single-strand NeoPixel + Servo FeatherWing + Prop-Maker outputs

Controller-to-controller link
- M0_SERIAL1_TX      -> RP2040_SERIAL1_RX
- M0_GND             -> RP2040_GND

Switch controls (momentary switches to GND, firmware uses INPUT_PULLUP)
- SW1: M0_PIN2 -> switch -> GND  (toggle on/off)
- SW2: M0_PIN3 -> switch -> GND  (previous sequence, 1->3 wrap)
- SW3: M0_PIN4 -> switch -> GND  (next sequence, 3->1 wrap)
- Hold SW1+SW3 for 5 seconds -> reset to sequence 1

M0 NeoPixel output (single strand)
- M0_PIN13 -> NeoPixel_Data (166-pixel SK6812 strip, Adafruit 4865)
- +3V7_NEO -> NeoPixel_Power
- GND_COMMON -> NeoPixel_Ground

RP2040 effect outputs
- RP2040_PIN5  -> R601 -> SPARK1_FILAMENT
- RP2040_PIN6  -> R602 -> SPARK2_FILAMENT
- RP2040_PIN9  -> R603 -> SPARK3_FILAMENT
- RP2040_PIN10 -> R604 -> SPARK4_FILAMENT
- RP2040_PIN12 -> PROPMAKER_LED1_CTRL
- RP2040_PIN13 -> PROPMAKER_LED2_CTRL
- RP2040_PIN14 -> PROPMAKER_LED3_CTRL
- RP2040_I2C_SDA -> SERVO_WING_SDA
- RP2040_I2C_SCL -> SERVO_WING_SCL
- SERVO_WING_CH0 -> SERVO_A_SIGNAL
- SERVO_WING_CH1 -> SERVO_B_SIGNAL
- RP2040_PIN8  -> PUMP_drain (optional, firmware disabled)

Power domains
- +5V_IN (from 5V battery via main fuse)
  +-> F801 (+5V_BUS direct)

- RP2040_BAT_IN (from dedicated PKCell LP503562 battery #1 at J811)
  +-> Feather RP2040 BAT input (built-in Feather charging path)
  +-> direct spark filament source domain

- +3V7_NEO_IN (from dedicated PKCell LP503562 battery #2 at J812)
  +-> +3V7_NEO (NeoPixel strip supply)

- CHG_NEO_USB_5V -> U812 (Adafruit 259) -> LP503562 battery #2 (J812)

- +5V_BUS -> Servo FeatherWing, Prop-Maker LED stage, peltier load domain, optional pump
- RP2040_BAT -> RP2040 controller supply and direct spark filament branch (Spark1-4)
- +3V7_NEO -> NeoPixel strip power domain

Direct spark configuration
- Spark outputs (R601-R604):
  - Direct from RP2040 GPIO pins 5, 6, 9, and 10
  - One 10 ohm series resistor per channel
  - Validate the selected filament current against RP2040 GPIO limits

- Prop-Maker LED outputs:
  - Control: RP2040 pins 12/13/14
  - Load side: 3-9W LED channels through the Prop-Maker FeatherWing output stage
  - Power domain: +5V_BUS

- Servo outputs:
  - Control bus: RP2040 I2C SDA/SCL
  - PWM generation: PCA9685 on Servo FeatherWing (address 0x40)
  - Outputs: CH0 -> Servo A, CH1 -> Servo B
  - Power domain: +5V_BUS

- Optional pump MOSFET:
  - Gate: RP2040 GPIO pin 8
  - Drain: Load negative
  - Source: GND_COMMON
  - Drain supply: +5V_BUS

Current-limiting resistors
- R601-R604: 10Ω series on spark loads
- R701-R703: 10Ω series on Prop-Maker LED channel paths if discrete output staging is retained

Fuses and ratings
- F801: 3A slow-blow (+5V_BUS main branch; all 5V loads peak ~2.5A)
- F802: 2A slow-blow (+3V_FILAMENT buck output; optional spark branch, verify final current)

Connector mapping
- J801: 5V power input connector (JST XT60 or barrel jack)
- J802: Power distribution hub (4-pin: +5V_BUS, RP2040_BAT, +3V7_NEO, GND)
- J811: 2-pin JST RP2040 main battery input (RP2040_BAT source, PKCell LP503562)
- J812: 2-pin JST NeoPixel battery input (+3V7_NEO source, PKCell LP503562)
- U812: Adafruit 259 charger for J812 NeoPixel battery only
- J601: 5-pin filament PWM bus (SPARK1-4, GND inputs from RP2040)
- J602: 2-pin JST filament 3V power
- J603: 4-pin filament outputs (post-resistor to loads)
- J701: 4-pin Prop-Maker control header (LED1/LED2/LED3 control + GND)
- J702: 4-position screw terminal for 3-9W LED output channels
- J401: 2-pin JST optional pump output
- J402: 3-pin Servo A connector (signal + 5V_BUS + GND)
- J403: 3-pin Servo B connector (signal + 5V_BUS + GND)

Grounding Rules
- [MANDATORY] Tie all grounds together:
  - RP2040 ground
  - M0 ground
  - LP503562 RP2040 battery negative
  - LP503562 NeoPixel battery negative
  - U812 charger ground
  - All MOSFET sources to GND_COMMON
  - NeoPixel strip ground
  - Pump/servo/LED driver grounds
  - Switch grounds
  - 5V input negative (battery pack or barrel jack shell)

Rail isolation rule
- Keep +5V_BUS, RP2040_BAT, and +3V7_NEO positive rails isolated from each other.
- Keep U812 charger output isolated to the NeoPixel battery rail only.

Firmware Implementation Notes
- RP2040 DeviceConfig.h: pump control remains optional on GPIO GP8, but firmware actions are currently commented out
- Servo control is now delegated to the PCA9685 on the Servo FeatherWing over I2C
- M0->RP2040 serial link remains unchanged at 40ms periodic EffectCommand updates
