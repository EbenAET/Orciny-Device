ORCINY KICAD NET MAP - RP2040 ONLY
==================================
Current electrical reference for the RP2040-only hardware build.

Controller and UI
- RP2040 is the only controller in this revision.
- SW1: RP2040_A1 (GP27) -> switch -> GND (toggle output on/off)
- SW2: RP2040_A2 (GP28) -> switch -> GND (previous sequence)
- SW3: RP2040_A3 (GP29) -> switch -> GND (next sequence)
- Hold SW1+SW3 for 5 seconds -> reset to sequence 1

NeoPixel core
- RP2040_GP4 -> NeoPixel_Data (Adafruit 4865, SK6812, 166 pixels)
- +3V7_NEO -> NeoPixel_Power
- GND_COMMON -> NeoPixel_Ground

RP2040 effect outputs
- RP2040_GP7  -> R601 -> SPARK1_FILAMENT
- RP2040_GP6  -> R602 -> SPARK2_FILAMENT
- RP2040_GP9  -> R603 -> SPARK3_FILAMENT
- RP2040_GP24 -> R604 -> SPARK4_FILAMENT
- RP2040_GP10 -> PROPMAKER_PWR_ENABLE (held HIGH)
- RP2040_GP11 -> PROPMAKER_LED1_CTRL
- RP2040_GP12 -> PROPMAKER_LED2_CTRL
- RP2040_GP13 -> PROPMAKER_LED3_CTRL
- RP2040_GP8 -> PELTIER_ENABLE_GATE (Q9)
- RP2040_I2C_SDA -> SERVO_WING_SDA
- RP2040_I2C_SCL -> SERVO_WING_SCL
- SERVO_WING_CH0 -> SERVO_A_SIGNAL
- SERVO_WING_CH1 -> SERVO_B_SIGNAL
- PUMP_ENABLE_GATE_OPTIONAL -> GPIO unassigned in this revision (optional, firmware disabled)

Power domains
- +5V_IN (from 5V battery via main fuse)
  +-> F801
  +-> +5V_BUS (ServoWing + Prop-Maker/peltier + optional pump)

- RP2040_BAT_IN (from dedicated LP503562 battery #1 at J811)
  +-> Feather RP2040 BAT input (Feather onboard charger path)
  +-> RP2040_BAT domain (controller and spark branch)

- +3V7_NEO_IN (from dedicated LP503562 battery #2 at J812)
  +-> +3V7_NEO (NeoPixel strip supply)

- CHG_NEO_USB_5V -> U812 (Adafruit 259) -> LP503562 battery #2 (J812)

Load domain summary
- +5V_BUS -> Servo FeatherWing, Prop-Maker LED stage, peltier source, optional pump
- RP2040_BAT -> RP2040 logic and direct spark filament branch
- +3V7_NEO -> NeoPixel strip only

Spark branch
- Spark channels are direct RP2040 GPIO outputs through 10 ohm series resistors.
- R601-R604: one 10 ohm resistor per spark channel.

Prop-Maker and servo domains
- Prop-Maker control inputs: GP11/GP12/GP13, with GP10 held HIGH for PWR enable
- Prop-Maker load supply domain: +5V_BUS
- Peltier control: GP8 drives Q9; firmware enables peltier while beam is active and holds peltier on briefly after beam off for heat dissipation
- ServoWing control: I2C SDA/SCL from RP2040
- ServoWing power: +5V_BUS and GND_COMMON

Optional pump branch
- Gate: GPIO unassigned in this revision (GP8 repurposed to peltier)
- Drain: load negative
- Source: GND_COMMON
- Load positive: +5V_BUS

Connector mapping
- J801: 5V power input connector (JST XT60 or barrel jack)
- J802: power distribution hub (+5V_BUS, RP2040_BAT, +3V7_NEO, GND)
- J811: RP2040 main battery input (LP503562 #1)
- J812: NeoPixel battery input (LP503562 #2)
- U812: Adafruit 259 charger for J812 NeoPixel battery only
- J601: spark PWM bus input (SPARK1-4 + GND from RP2040)
- J603: spark load outputs (post-resistor)
- J701: Prop-Maker control header (LED1/LED2/LED3 control + GND)
- J702: 3-9W LED output connector
- J703: peltier load output
- J401: optional pump output
- J402: Servo A connector (signal + +5V_BUS + GND)
- J403: Servo B connector (signal + +5V_BUS + GND)

Grounding rules
- [MANDATORY] Tie all grounds together:
  - RP2040 ground
  - LP503562 RP2040 battery negative
  - LP503562 NeoPixel battery negative
  - U812 charger ground
  - NeoPixel strip ground
  - Spark returns
  - Servo/Prop-Maker/pump/peltier returns
  - Switch grounds
  - 5V input negative

Rail isolation rules
- Keep +5V_BUS, RP2040_BAT, and +3V7_NEO positive rails isolated from each other.
- Keep U812 charger output isolated to the NeoPixel battery rail only.
