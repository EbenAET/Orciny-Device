ORCINY DEVICE - ASCII CIRCUIT DIAGRAM
======================================
Rev F  (RP2040-only control; split battery domains)

Harness cross-reference: see `circuit/orciny_wiring_harness_reference.md`.

Power:
  Harness IDs: H-PWR-5V, H-BAT-RP, H-BAT-NEO
  5V battery (J801)  -> main fused input -> +5V_MAIN trunk
  +5V_MAIN trunk     -+-> F801 (direct) -------- +5V_BUS rail (ServoWing + Prop-Maker/peltier load domain)
  3.7V battery (J811, PKCell LP503562 1200mAh) -> Feather RP2040 BAT input (controller + spark domain)
  NeoPixel strip supply (current profile): +5V_BUS rail
  Legacy optional path: Adafruit 259 charger (U812, CHG_NEO) -> LP503562 battery at J812 -> +3V7_NEO rail

  +5V_BUS rail       -> Servo FeatherWing/load supply, Prop-Maker LED load supply, peltier source domain
  RP2040_BAT rail    -> RP2040 logic supply and direct spark filament source domain
  +3V7_NEO rail      -> legacy optional NeoPixel-only rail
  GND_COMMON         -> COMMON to every board, switch, driver, and load

  Note: if legacy +3V7_NEO is used, only the NeoPixel battery uses external Adafruit 259 charging; the RP2040 battery uses Feather onboard charging.

 RP2040 A1(GP27)    RP2040 A2(GP28)    RP2040 A3(GP29)
   GND               GND               GND
   Hold SW1 + SW3 together for 5s -> reset to sequence 1


 Direct Spark Filament Stage:
 Harness ID: H-SPARK

   RP2040 GPIO pin
         |
   [10 ohm series resistor]
         |
   Spark filament channel

  Spark branch is supplied from RP2040_BAT in this revision.


 MOSFET Driver Stage (beam / peltier / optional pump paths):
 Harness IDs: H-BEAM-PWR, H-PELTIER-PWR, H-PUMP-OPT
 (GP11/12/13 beam control, GP10 PWR enable, and GP8 peltier control route through FeatherWing stack headers; not separately harnessed)

  Firmware thermal behavior:
    GP8 drives Q9 for peltier control.
    Peltier turns ON while beam is enabled and remains ON briefly after beam turns OFF for heat dissipation.

   RP2040 GPIO pin
         |
     (gate)  N-channel MOSFET (logic-level, Vgs(th) < 2.5V)
         |
   drain -- Load negative terminal
   source -- GND_COMMON

  Load positive terminal -- +5V_BUS (beam/peltier domain)
  Optional pump load positive terminal -- +5V_BUS


 Spark Filament Channels (J601 -> R601-R604 -> J603):
 Harness ID: H-SPARK

  RP2040_BAT domain -> spark supply source (direct GPIO channels)
   J601 PWM bus -> R601 10Ω -> Spark Filament 1 load
   J601 PWM bus -> R602 10Ω -> Spark Filament 2 load
   J601 PWM bus -> R603 10Ω -> Spark Filament 3 load
   J601 PWM bus -> R604 10Ω -> Spark Filament 4 load


 Prop-Maker LED Outputs (3-9W LED):
 Harness ID: H-BEAM-PWR
 (GP11/12/13 control signals and GP10 PWR enable enter Prop-Maker via stacked header; not a separate cable harness)

  +5V_BUS powers the Prop-Maker LED output stage
  RP2040 GP11/12/13 provide control for LED1/LED2/LED3 channels
  RP2040 GP10 must be held HIGH to enable Prop-Maker LED outputs
  LED load returns to GND_COMMON

  Peltier load is sourced from the Prop-Maker 5V supply domain.


 Pump Channel (optional):
 Harness ID: H-PUMP-OPT

  Pump MOSFET gate is unassigned in this revision (GP8 repurposed to peltier), and firmware actions remain disabled.


 Power Domain Summary:
   Rail              Source               Protected By   Consumers
   +5V_BUS           Direct from 5V bat   F801 (3A)      Servo FeatherWing + servos, Prop-Maker load stage, peltier, optional pump
   RP2040_BAT        LP503562 cell #1     Feather onboard controller supply, direct spark filament branch (Spark 1-4)
  CHG_NEO           Adafruit 259 (U812)  Charger IC     Legacy optional Neo battery path only
  +3V7_NEO          LP503562 cell #2     Legacy optional NeoPixel-only rail


 Wire Gauge Guidance (high-current runs):
   +5V_MAIN and +5V_BUS trunk wiring: use 18 AWG minimum.
  High-current branch feeds and returns (+5V_BUS to NeoPixel/servo wing/Prop-Maker/peltier, RP2040_BAT spark branch): use 20 AWG minimum.
   Keep GND return wiring for each high-current branch at the same gauge as its positive feed.
   Signal/control lines (GPIO, I2C, serial, switch leads): 24-26 AWG is acceptable.
   If a branch run is long or heating is observed, step one gauge larger and re-validate thermal rise under peak load.
   Use a local inline fuse at each LP503562 battery positive lead near the cell.


 Quick Net Summary:
   Harness map:
   H-PWR-5V    -> +5V input trunk and +5V_BUS load backbone
   H-BAT-RP    -> J811 RP2040 BAT battery run
  H-BAT-NEO   -> optional legacy J812 NeoPixel battery and U812 charger pair
   H-CORE      -> NeoPixel data/power/ground run
   H-SW        -> switch inputs to controller
   H-SPARK     -> spark PWM channels and returns
  H-SERVO     -> ServoWing outputs and power/ground branches (I2C enters via stacked header)
  H-BEAM-PWR  -> +5V_BUS beam load-side feed
  H-PELTIER-PWR -> +5V_BUS peltier load-side feed
   H-PUMP-OPT  -> optional pump control/load branch

   RP2040 A1(GP27) <- SW1 -> GND   (toggle ON/OFF)
   RP2040 A2(GP28) <- SW2 -> GND   (previous sequence, 1->3 wrap)
   RP2040 A3(GP29) <- SW3 -> GND   (next sequence, 3->1 wrap)
  RP2040 GP25  -> NeoPixel strip data (166 pixels, SK6812, Adafruit 4865, single strand)
   J811         -> RP2040 Feather BAT input (main 3.7V controller/spark battery)
  U812         -> optional legacy charger for J812 LP503562 battery (+3V7_NEO)
  +5V_BUS      -> NeoPixel strip +V (current profile)
   +5V_BUS      -> Prop-Maker load supply and peltier source domain
  RP2040 GP7/GP6/GP9/GP24 -> direct spark filament outputs (R601-R604)
  RP2040 GP10      -> Prop-Maker PWR enable (held HIGH)
  RP2040 GP11-GP13 -> Prop-Maker LED1/LED2/LED3 channel control
  RP2040 GP8       -> peltier MOSFET gate (auto beam-follow + cooldown hold)
   RP2040 SDA/SCL   -> Servo FeatherWing (PCA9685 @ 0x40)
   Servo Wing CH0/1 -> Servo A / Servo B signal lines
   Pump GPIO        -> unassigned in this revision (GP8 reassigned to peltier)
   Hold SW1 + SW3 for 5s => reset to sequence 1
